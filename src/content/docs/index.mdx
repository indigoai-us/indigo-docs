Indigo HQ is the operating system for AI workers — a shared context layer over Claude Code, Cursor, and Codex that syncs knowledge, skills, policies, and capabilities across your team. It scales from a solo builder to an enterprise.

## Get started

```bash
npx create-hq
cd hq && claude
/setup
```

Then dive into the docs:

- [What is HQ?](/hq/guide/1-what-is-hq/) — the product, the model, and why it exists.
- [Getting started](/hq/guide/2-getting-started/) — install, set up, and run your first session.
- [Core concepts](/hq/guide/3-core-concepts/) — companies, workers, knowledge, policies, projects.
- [Architecture](/hq/architecture/1-overview/) — how HQ is built under the hood.

For a high-level tour of the broader HQ platform, see the [HQ Platform docs](https://github.com/indigoai-us).

## Meetings & signals

Meeting intelligence and signals now live inside HQ as company skills and a synced signals store, surfaced through the `hq` CLI and HQ workers. This capability is being rebuilt on HQ — documentation will return here once it ships.
